# SpeakUp — AI Public Speaking Coach
## STRUCTURE Document v2

---

## Vision

A personal AI coach that fixes the root problem first: **you don't have the vocabulary**.
Learn phrases and sentence patterns from Vinh Giang's content, then practice with AI feedback.
Feels like a coach with a plan — not a toolbox.

Contexts covered: Job Interviews, Church Prayer, Church Announcements, Presentations & Pitches,
Casual Storytelling, General Public Speaking.

---

## 3-Layer Architecture (Claude Code Framework)

### Layer 1: Directive — `directives/`
Markdown SOPs telling Claude what to do and why.

### Layer 2: Orchestration — Next.js API routes + Claude
Reads directives, routes to execution scripts, handles errors.

### Layer 3: Execution — `execution/`
Deterministic Python scripts doing the actual API work.

## The Five Screens

### 1. Dashboard — "Your Coach" (app/page.tsx)

Not a menu. A daily briefing.

- Greeting + streak badge
- **Today's Focus card** — one specific coach-assigned task
  - "Today: Learn 5 phrases for opening a presentation"
  - "Today: Practice your job interview answer for Tell me about yourself"
- 6 small progress rings (one per context)
- Recent activity (last 3 actions)
- "Continue" button → today's task

Coach Plan generated by `execution/generate_plan.py`:
- Reads userProgress from localStorage
- Builds a 7-day plan prioritising weakest contexts
- Refreshes daily, stored in localStorage

---

### 2. Learn — "Build Your Vocabulary" (app/learn/)

Powered entirely by `data/vinh_giang.json` via `contentLibrary.ts`.

**Context Selector (app/learn/page.tsx):**
Six context cards:
- 💼 Job Interviews
- 🙏 Church Prayer
- 📢 Church Announcements
- 🎤 Presentations & Pitches
- 💬 Casual Storytelling
- 🗣️ General Public Speaking

Each shows % learned and item count.

**Topic Detail (app/learn/[topic]/page.tsx) — 3 tabs:**

Tab 1 — Phrases
- PhraseCards from getPhrasesByTopic(topic)
- Each: phrase, when to use it, source attribution
- Actions: Save / Learned / Copy
- Swipeable on mobile

Tab 2 — Vocabulary
- VocabCards from getVocabularyByTopic(topic)
- Each: word, meaning, example sentence
- Same Save / Learned / Copy actions

Tab 3 — Sentence Builder
- User types a rough idea
- SentenceBuilder calls Claude with:
  - The rough idea
  - Relevant Vinh phrases/vocab for that topic
  - Instruction: return 5 versions simple → powerful
- Returns 5 alternatives labelled: Simple / Clear / Confident / Polished / Powerful
- Copy any version

Every "Learned" tap updates userProgress in localStorage.

---

### 3. Practice — "Use What You've Learned" (app/practice/page.tsx)

Two sub-modes on one screen.

**Sub-mode A: Live Practice**
- Scenario: Job Interview / Pitch / Storytelling / Open
- AI asks questions (text + SpeechSynthesis)
- User responds via microphone (Web Speech API)
- Per-turn scorecard: pace, fillers, confidence 1-10
- Vocabulary suggestion: pulls from user's saved phrases specifically
- Max 10 turns → full session scorecard

**Sub-mode B: Recording Review**
- Record in browser or upload audio
- Whisper API transcription via app/api/analyse/
- Full scorecard + vocabulary upgrade panel
- Vocabulary upgrades pull from vinh_giang.json

**ScoreCard output (both sub-modes):**
- WPM (green 120-160, amber outside, red >200)
- Filler words highlighted
- Clarity 1-10
- Vocabulary richness 1-10
- Top 3 tips
- One highlight (always positive)
- Vocabulary upgrades: "You said X. Try instead: Y (Vinh Giang)"

---

### 4. Prayer — "Speak with Confidence at Church" (app/prayer/page.tsx)

Standalone, warm gold accent. Four tabs.

**Tab 1 — Scenario**
Six scenarios from prayerScenarios.ts → AI sets scene → user speaks/types → PrayerScoreCard.

PrayerScoreCard:
- Structure: Address / Acknowledgement / Intercession / Close (tick or circle each)
- "Just" filler count (red if >3)
- Standard fillers
- Length: Too Short / Ideal / Too Long
- Warmth 1-10
- Clarity 1-10
- Top 3 tips + one highlight

**Tab 2 — Recording**
Record/upload → PrayerScoreCard output.

**Tab 3 — Phrase Bank**
Categories: Opening Addresses, Acknowledgement Lines, Transitions, Strong Closings, Announcement Openers.
Sourced from vinh_giang.json prayer_speaking topic, fallback to hardcoded.
Copy button. Disclaimer: "Inspiration, not scripts."

**Tab 4 — Tips**
Static written tips per scenario type drawn from Vinh content + hardcoded church coaching principles.
No YouTube feed.

---

### 5. Progress (app/progress/page.tsx)

- Current streak
- 6 progress rings (phrases learned / total per context)
- Total phrases saved
- Total practice sessions
- Total prayers practiced
- "Your weakest area" with nudge to practice
- All from localStorage via userProgress.ts

## Claude API Prompts

### Sentence Builder
```
System: You are a public speaking coach trained in Vinh Giang's methodology.
The user wants to express an idea in a [context] context.
Relevant phrases they are studying: [inject from contentLibrary]
Show 5 ways to express their idea, simple to powerful.
Label each: Simple / Clear / Confident / Polished / Powerful.
Return ONLY JSON array:
[{ "level": "Simple", "sentence": "...", "note": "why this works in one phrase" }]
```

### Live Practice per-turn
```
System: You are a speech coach. Analyse this spoken response in a [scenario] context.
User's saved phrases: [inject from userProgress].
Return JSON only:
{
  "pace": "slow|good|fast",
  "fillerCount": N,
  "confidenceScore": 1-10,
  "tip": "one specific tip under 20 words",
  "praise": "one genuine praise under 20 words",
  "vocabularyUpgrade": { "theyUsed": "...", "tryInstead": "..." } | null
}
```

### Recording Review
```
System: You are a professional speech coach.
Analyse this transcript from a [scenario] session.
User's saved phrases: [inject from userProgress].
Return JSON: wpm estimate, filler word list, structure score 1-10,
clarity score 1-10, vocabulary richness 1-10, top 3 tips, one highlight,
vocabularyUpgrades: [{ theyUsed, tryInstead, source: "Vinh Giang" }]
```

### Prayer Analysis
```
System: You are a compassionate but honest church speaking coach.
Return JSON:
{
  "structureScore": 1-10,
  "structureParts": { "address": bool, "acknowledgement": bool, "intercession": bool, "close": bool },
  "justFillerCount": N,
  "fillerWordList": [...],
  "lengthRating": "too_short|ideal|too_long",
  "warmthScore": 1-10,
  "clarityScore": 1-10,
  "tips": ["...", "...", "..."],
  "highlight": "..."
}
```

## Tech Stack

| Layer | Choice | Reason |
|---|---|---|
| Framework | Next.js 14 App Router | Consistent with other projects |
| Language | TypeScript strict | Type safety |
| Styling | Tailwind CSS | Mobile-first |
| Content | data/vinh_giang.json | Pre-built, no live API for content |
| Transcription | OpenAI Whisper API | Best accuracy |
| AI | Anthropic claude-sonnet-4-20250514 | Analysis + conversation |
| Live Speech | Web Speech API | Free, browser native |
| Progress | localStorage | No backend needed |
| Hosting | Vercel | Free tier |
